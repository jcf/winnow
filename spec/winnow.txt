# Winnow Conformance Spec
# ======================
# Format:
#   - Lines starting with # are comments
#   - Non-#, non-blank, non-=> lines are input classes
#   - Multiple input strings separated by " | "
#   - => lines are expected output
#   - Blank lines separate test cases visually
#
# Compatibility:
#   Based on tailwind-merge v3.x test cases as a starting point.
#
#   INTENTIONAL DIVERGENCE (see "Unknown classes" section near end of file):
#   - Unknown values pass through unchanged rather than being treated as colors
#   - Winnow requires explicit configuration for custom colors via make-resolver
#   - This is safer for multi-framework projects and catches typos
#
#   See CLAUDE.md "Behavioral Differences" section for full rationale.
